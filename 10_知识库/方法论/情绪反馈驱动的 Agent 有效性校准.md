# 情绪反馈驱动的 Agent 有效性校准

## 解决什么问题

Agent 常常只看任务是否完成，却忽略用户反馈里的有效性信号。

用户说“可以”“good”“对喽”“流程对了”时，不一定代表最终产物完美，而是说明当前方向、粒度或协作方式开始有效。

用户说“不够”“太浅”“没有干货”“没有关联”“比例不好看”“为什么之前搞不定”时，也不只是情绪，而是在指出输出和真实目标之间的偏差。

## 核心结论

用户情绪不是要被安抚的噪音，而是 agent 校准工作方式的证据。

下一次相似任务里，agent 应该把情绪反馈转成可执行判断：继续、收窄、停下、换证据、改流程、降自动化、补原文、重做结构。

## 反馈信号表

### 正向信号

- “可以”“继续”：方向基本可行，可以推进，但仍需保持验证。
- “good”：当前动作命中了用户的真实关心点，适合扩展到相邻对象。
- “对，就是这个思路”：上位结构成立，应把它固化成方法或模板。
- “对喽，这样的东西才有价值”：前面曾经偏离目标，当前才接近可复用价值。
- “流程对了”：流程骨架可保留，下一步优化效率和稳定性。

### 负向信号

- “不够”“太浅”“没有干货”：信息源或抽象层级不足，不应继续填库。
- “没有关联”“不相关”：证据和目标错位，必须回到素材、场景和功能点的对应关系。
- “不好看”“比例不对”“排版不好”：视觉外壳没有承载内容，先调结构而不是补文案。
- “你经常把内容发送错位置”“一直在瞎跳转”：自动化流程缺少状态判断，不能继续盲跑。
- “我让你和脚本协同，不是让我和脚本协同”：agent 不能把监督成本转嫁给用户。
- “先停”：当前方向已经伤害效率，应停止扩展，保留事实和边界。

## 使用方法

每次任务结束后，先不要问“做完了吗”，而要问：

- 用户的最后三次反馈是在鼓励推进，还是在纠偏？
- 纠偏是内容层、证据层、视觉层、流程层，还是协作层？
- 用户是否在重复同一个负面信号？
- 当前产物是否因为用户纠偏才变得有效？
- 这个纠偏能不能变成下次 agent 的前置约束？

如果负面信号重复出现，优先改流程，不要继续堆产物。

## 判断动作

- 用户嫌浅：回到原文、样本、日志、视频对白或实验记录。
- 用户嫌乱：先重构目录、命名和信息架构。
- 用户嫌假：拆分真实证据和生成表达。
- 用户嫌慢：减少探索空间，固化可重复流程。
- 用户嫌不懂意图：暂停执行，用一句话复述目标和边界。

## 反模式

- 把“可以”误解成完全满意。
- 把“继续”当成不需要验证。
- 用户已经指出浅，还继续写漂亮总结。
- 用户指出流程错，还继续自动化盲跑。
- 用户指出素材不相关，还继续优化版式。
- 用户中断后，agent 不复盘中断原因。

## 证据成熟度

成熟度：待复用验证。

证据来自 2026-06-29 至 2026-06-30 的多类任务：抖音内容采集、地铁功能图重做、小学生课程设计和 GitHub 同步。它们共同说明，用户的短句反馈能稳定暴露有效性边界。
