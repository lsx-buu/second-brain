# 实验结果转成可迁移判断

## 核心问题

实验结果本身通常不可迁移。

可迁移的是实验结果背后的判断：为什么这个结果可信，为什么这个结果不能用，为什么这个方向该停，为什么另一个方向值得继续。

## 判断模板

看到一个实验结果时，不先问“数字多少”，先问六个问题：

- 它和什么基线比。
- 它在哪个评价口径下成立。
- 它是否只在验证集成立。
- 它是否违反真实约束。
- 它的收益来自稳定机制，还是来自偶然切分。
- 它能改变下一步决策吗。

如果不能改变下一步决策，这个结果通常不值得单独成卡。

## 结果的四种命运

第一种：升级为方法。

条件是它在多个场景成立，并能形成可复用判断。例如防未来函数审计优先于收益优化。

第二种：保留为反例。

条件是它解释了一个常见误区。例如 validation 好看但 test 不成立。

第三种：降级为项目事实。

条件是它只对当前项目有局部意义。例如某个版本具体分数。

第四种：丢弃。

条件是它既不影响决策，也不能形成判断。

## 从结果到方法的句式

不要写：

某模型 test 没超过某 baseline。

要写：

当结构更干净但不能跨切分稳定超过 baseline 时，它适合作为 negative evidence，而不是主方法。

不要写：

某策略收益从 -31% 到 -10%。

要写：

从全池模型收窄到可解释 setup 可以减少伤害，但防守改善不等于可交易 edge。

不要写：

某 oracle gain 很高。

要写：

oracle 上界只说明理论空间，不说明当前可见信息能学到；必须再做可学习性审计。
