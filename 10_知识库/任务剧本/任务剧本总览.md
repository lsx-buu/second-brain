# 任务剧本总览

## 这层解决什么问题

方法卡告诉 agent 为什么这样做。

任务剧本告诉 agent 开工时先做什么、不要做什么、怎样判断是否跑偏。

当任务明确、需要落地执行时，优先读任务剧本；当任务复杂、需要理解原理时，再读关联方法。

## 使用顺序

1. 先读 [[90_索引/Agent 启动上下文|Agent 启动上下文]]。
2. 再按任务类型进入对应剧本。
3. 剧本不够用时，再读关联方法群和方法卡。
4. 任务结束后回到剧本的“写回判断”，决定是否沉淀。

## 写法规则

每张任务剧本都尽量包含：

- 什么时候读。
- 开工前先看什么。
- 第一轮动作。
- 不要做什么。
- 有效信号。
- 失败信号。
- 写回判断。

## 不要这样用

不要把任务剧本写成完整教程。

不要在剧本里塞路径、脚本名和一次性现场状态。

不要让剧本替代方法卡。剧本只负责启动动作和边界判断。
