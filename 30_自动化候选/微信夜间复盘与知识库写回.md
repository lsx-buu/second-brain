# 微信夜间复盘与知识库写回

## 想自动化什么

通过 cc-weixin 把夜间复盘放到微信里完成。

白天工作对话不显式导入材料，只把明显有价值的内容放进 [[00_收件箱/夜间复盘候选|夜间复盘候选]]。

晚上用户在微信里发送“复盘”，Agent 读取候选，用轻量问题完成复盘，并在结束后把材料整理进入知识库。

## 为什么值得做

用户不希望工作对话被复盘打断，也不希望“导入新材料”变成一个显式任务。

微信复盘的优势是低摩擦：晚上只需要像聊天一样回答一句话或三问，Agent 再负责整理、归档和写回。

## 最小可用版本

- cc-weixin 服务常驻。
- 用户晚上在微信发送“复盘”。
- Agent 读取夜间复盘候选。
- Agent 只问一个轻问题开始。
- 复盘结束后，Agent 按新内容导入 SOP 写回知识库。

## 后续增强版本

拿到微信 chat_id 并确认权限后，可以再考虑主动提醒。

主动提醒必须满足：

- 只提醒白名单用户。
- 默认每天最多一次。
- 可以随时关闭。
- 不包含敏感内容。
- 不直接发送完整候选材料，只发轻量开场。

## 风险边界

不要把微信变成另一个噪音入口。

不要把完整微信对话写入长期知识库。

不要在未确认 chat_id 和权限前主动推送。

不要在工作对话里强行复盘。
