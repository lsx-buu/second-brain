# 自动生成 Agent 启动上下文与交接摘要

## 想自动化什么

让 Codex 或其他 coding agent 在新对话、新线程或任务中断后，自动获得一份短上下文。

这份上下文不是完整聊天记录，而是下一位 agent 继续工作真正需要的信息。

## 为什么值得自动化

Agent 的主要限制是上下文长度和对话隔离。

如果每次都从零解释用户偏好、项目状态、方法选择和已做判断，协作成本会很高。

自动生成启动上下文和交接摘要，可以让 agent 越用越懂用户，而不是每个线程都重新开始。

## 输入材料

- 当前任务目标。
- 用户最后确认的方向。
- 已读取的知识卡。
- 已完成的操作。
- 未完成事项。
- 关键文件、项目或外部材料。
- 需要写回知识库的判断。

## 期望输出

- 当前目标。
- 用户真实意图。
- 已完成事项。
- 未完成事项。
- 应继续读取的知识卡。
- 下一位 agent 第一件事。
- 不要重复做的事情。
- 是否需要更新知识库。

## 触发时机

- 长任务即将中断。
- 对话上下文快满。
- 用户要求继续到新线程。
- agent 完成一阶段工作。
- 需要把任务交给另一个 coding agent。

## 人需要审核什么

- 是否遗漏关键约束。
- 是否误解用户真实意图。
- 是否包含不该长期保存的敏感信息。
- 是否把临时事实写成长期规则。
- 是否能让下一位 agent 直接接手。

## 风险和边界

交接摘要不能替代知识库。

它只保存当前任务的接力信息；长期方法、偏好和成熟度仍应写回对应知识卡。

## 可先手动验证的最小版本

每次长任务结束前，使用 [[_模板/Agent 任务交接模板|Agent 任务交接模板]] 生成一张交接摘要。

连续成功使用几次后，再考虑自动生成。
