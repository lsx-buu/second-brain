# 2026-06-27 知识库质量审计

## 一句话结论

当前知识库已经完成了“从无到有”“初步中文化”和“方法论纠偏”，并开始进入压缩升维阶段。

它曾经一度被整理成“项目接管手册”，这被确认是偏离目标的。真正目标是：把项目当材料，把可迁移判断、反模式和方法论当长期资产。

第四轮进一步确认：节点数量不是越多越好。如果一组兄弟节点结构相同，只是换了功能名、模型名或项目名，它们应该合并成更深的母卡。

这些指标说明：链接健康，但标签系统明显发散；卡片数量看起来不少，但部分卡片还没有形成网络。

标签从 98 个压到 33 个，单次标签比例从 72.4% 降到 18.2%。这说明标签已经从临时关键词，开始变成可复用分类系统。

服务器项目曾经从 3 张卡扩展成运行手册、实验索引、决策日志、风险边界和下一步队列，但这些卡片被确认价值不足：它们更像项目现场管理，不像第二大脑里的方法论资产。

新的质量判断标准：

如果原项目文件被删除，这张卡仍然有用，才算有效知识。

如果一张卡只能告诉我“去哪看文件、哪个实验叫什么、某个模型分数是多少”，它应该降级为临时项目记录，不应作为长期知识节点。

更高一层的第一性原理是：

这些东西，对我下一次同样类型的、或者有一定相似性的项目，能不能产生参考价值和指导意义？

不能产生迁移价值的内容，不进入长期知识库。

仍然没有解决的问题是：少数说明卡、模板卡和入口卡仍然无出链，后续应区分“天然不需要强链接的系统文件”和“应该被合并或改写的浅卡”。

压缩前，AI 原型方法分成 MVP、可见原型、互动闭环和四层拆解；压缩后，它们成为一条从想法到可验证体验的流程。

压缩前，TS-CLIP 和 watchbot_online 各自散落多张审计卡；压缩后，它们分别沉淀成模型可靠性审计和实验收益可信性审计。

导入新内容时，下一次 agent 不应该直接总结，而应该先判断它的命运：丢弃、暂存、补旧卡、新建卡，还是生成复盘。

成长型任务中，下一次 agent 不应该默认当答案机器，而应该先帮助用户形成初始判断，再提问、挑战、反馈和复盘。

后续修正：复盘不应插入工作对话。工作线程只负责推进和交付，最多留下复盘候选；真正复盘应由用户单独触发，并默认采用一句话或三问的轻量形式。

## 怎样量化“项目是否沉淀够深”

对 watchbot_online 和 TS-CLIP 这种项目，不应该只看字数、文件数或现场完整度。应该按“方法论有效性”打分。

第三轮后，每个重要项目不再要求 6 类项目档案卡，而是至少回答 5 个方法论问题：

- 这个项目暴露了什么普遍问题？
- 这个项目里哪些事实能变成判断？
- 哪些判断能变成方法？
- 哪些内容离开项目就失效，应该丢掉？
- 下次遇到类似项目，应该如何复用这套方法？

第三轮后的当前状态：

- watchbot_online：已有项目深挖和方法论提炼，核心方法是“实验收益可信性审计方法”。
- TS-CLIP：已有项目深挖和方法论提炼，核心方法是“模型上界与可观测性审计方法”。

第一轮服务器项目方法论深度约 35 / 100。

第二轮服务器项目结构深度约 72 / 100，但方法论有效性不足。

第四轮服务器项目方法论深度约 84 / 100。

当前方向正确，但还需要继续把项目事实压缩成判断清单、反模式和迁移边界，并让每张方法论卡被多个项目反复调用。

## 推荐质量仪表盘

以后每次沉淀后，都看这 9 个指标：

- 入口覆盖率：每个重要项目是否能从首页三步内到达。
- 断链数：必须为 0。
- 无出链比例：目标低于 25%。
- 单次标签比例：目标低于 40%。
- 重要项目提炼度：是否能从项目事实提炼出可迁移判断、反模式和方法论。
- 方法论复用度：每张核心方法论卡至少被 2 个项目或案例引用。
- 同质节点压缩率：结构相同的兄弟节点是否被合并到上位母卡。
- 可迁移判断率：新增卡片是否能离开原项目仍然指导行动。
- 原始记录压缩率：保留判断和结构，不保存整段聊天和代码。

## 下一轮改进优先级

第一优先级：验证 Agent 启动上下文是否真的能成为每次任务的实际入口。

后续新线程或新 agent 接手时，优先让它读取 [[90_索引/Agent 启动上下文|Agent 启动上下文]]，并观察是否能减少重复解释。

第二优先级：把新内容导入和 AI 反哺复盘变成半自动检查清单。

下一步可以让每次沉淀自动生成一份检查结果：材料命运是什么、是否命中方法群、是否命中复用验证队列、建议补哪张旧卡、是否需要新建卡、是否需要生成 AI 反哺记录。

第三优先级：给单项目强证据方法补第二案例。

优先补 [[10_知识库/方法论/从需求文档提炼功能字段和硬件清单|从需求文档提炼功能字段和硬件清单]] 与 [[10_知识库/方法论/汇报型视觉材料迭代方法|汇报型视觉材料迭代方法]] 的非地铁案例。

第四优先级：验证待复用方法。

下一次出现学习、备考、技能训练、论文阅读陪跑或能力成长任务时，优先验证 [[10_知识库/方法论/从零备考的陪跑式学习方法|从零备考的陪跑式学习方法]] 与 [[10_知识库/方法论/AI 反哺式学习与问题解决|AI 反哺式学习与问题解决]]。

第五优先级：继续检查方法群是否真的内聚。

如果一个方法群下面的卡片只是被硬塞进去，而不是回答同一个上位问题，就继续拆分或归位。

第六优先级：继续检查入口是否按问题进入。

每次新增卡片前先问：它应该出现在首页的问题入口里，还是只应该补进某张母卡？

第七优先级：继续压缩同质兄弟节点。

如果一组卡只是换了功能名、项目名或模型名，优先合并成上位母卡。

第八优先级：把早期方法论卡改成真正的方法论。

每张方法论卡都补齐：适用场景、不适用场景、判断清单、反模式、迁移边界。

第九优先级：给自动沉淀加质量门禁。

每次自动写卡时，必须检查：是否有来源、是否有链接、是否有下一步、是否复用了稳定标签、是否补充了已有卡而不是新造浅卡。
