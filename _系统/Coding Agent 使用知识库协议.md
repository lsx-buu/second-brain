# Coding Agent 使用知识库协议

## 核心定位

这个知识库主要服务 Codex 或其他 coding agent。

它不是给人慢慢阅读的资料库，而是用来解决 agent 的几个长期问题：

- 单次上下文长度有限。
- 不同对话之间互相隔离。
- 用户偏好需要反复解释。
- 项目经验容易丢失。
- 失败模式和判断标准无法跨线程复用。

## Agent 使用目标

Agent 使用这个知识库时，应优先完成五件事：

- 快速恢复用户偏好。
- 快速判断当前任务属于哪类问题。
- 调用已有方法，而不是重新发明流程。
- 复用项目案例里的判断边界。
- 任务结束后把新经验写回知识库。

## 启动时先读什么

新对话或新 agent 接手时，先读 [[90_索引/Agent 启动上下文|Agent 启动上下文]]。

不要一开始扫描整个知识库。

如果当前 agent 在服务器或远程环境里运行，应先确认知识库是否通过 GitHub 同步，规则见 [[_系统/GitHub 同步与服务端读取规则|GitHub 同步与服务端读取规则]]。

如果任务明确，再按启动上下文里的路由读取相关方法群、方法卡和项目案例。

## 上下文预算原则

Agent 不应该把整个 vault 塞进上下文。

正确顺序是：

- 先读启动上下文。
- 再读用户偏好。
- 再读问题路由。
- 再读任务命中的方法群。
- 最后只读必要项目案例。

如果上下文不够，优先保留：

- 当前任务目标。
- 用户偏好。
- 任务相关方法卡。
- 关键项目案例。
- 写回规则。

可以丢弃：

- 完整历史聊天。
- 与当前任务无关的方法群。
- 只对旧项目现场有效的细节。

## 执行时怎么用

Agent 处理任务时，不要只引用知识库内容。

要把知识库变成行动：

- 遇到 AI 原型任务，调用 [[10_知识库/方法论/方法群/AI 原型验证方法群|AI 原型验证方法群]]。
- 遇到汇报材料任务，调用 [[10_知识库/方法论/方法群/业务可信表达方法群|业务可信表达方法群]]。
- 遇到实验或模型结果，调用 [[10_知识库/方法论/方法群/实验与模型审计方法群|实验与模型审计方法群]]。
- 遇到环境、接口、部署问题，调用 [[10_知识库/方法论/方法群/工程运行方法群|工程运行方法群]]。
- 遇到知识沉淀任务，调用 [[10_知识库/方法论/方法群/知识沉淀方法群|知识沉淀方法群]]。
- 遇到学习、偏好和长期协作任务，调用 [[10_知识库/方法论/方法群/学习与协作方法群|学习与协作方法群]]。
- 遇到“让 AI 反哺我、训练我、复盘我”的任务，调用 [[10_知识库/方法论/AI 反哺式学习与问题解决|AI 反哺式学习与问题解决]]，并尽量让用户先形成自己的判断。

## 结束时怎么回写

任务结束后，Agent 应判断是否需要写回知识库。

优先顺序：

- 补旧方法卡的证据。
- 更新项目案例。
- 更新偏好。
- 更新方法复用验证队列。
- 必要时新建卡。

新建卡永远不是默认动作。

如果当前是工作对话，不要强行展开复盘。可以内部判断是否有复盘候选，真正复盘应等用户单独触发。

写回前必须先看：

- [[_系统/Codex 对话沉淀 SOP|Codex 对话沉淀 SOP]]
- [[_系统/新内容导入 SOP|新内容导入 SOP]]
- [[_系统/AI 反哺复盘 SOP|AI 反哺复盘 SOP]]
- [[_系统/GitHub 同步与服务端读取规则|GitHub 同步与服务端读取规则]]
- [[90_索引/方法复用验证队列|方法复用验证队列]]
- [[90_索引/方法成熟度与证据索引|方法成熟度与证据索引]]

## Agent 交接原则

如果一个任务跨线程、跨 agent 或跨天继续，必须写清楚：

- 当前目标。
- 已完成什么。
- 还没完成什么。
- 读过哪些知识卡。
- 做过哪些判断。
- 哪些文件或项目是关键上下文。
- 下一位 agent 第一件事应该做什么。

交接应使用 [[_模板/Agent 任务交接模板|Agent 任务交接模板]]。

## 禁止事项

- 不要把知识库当项目现场目录。
- 不要保存完整聊天记录。
- 不要把代码片段当方法论卡核心。
- 不要为了显得完整而读取无关卡片。
- 不要在没有第二案例时过度泛化方法。
- 不要绕过用户偏好直接套通用流程。
