# 新内容导入 SOP

## 目标

把新内容导入第二大脑，但不让知识库变成资料仓库。

新内容可以是对话、文档、网页、图片说明、项目材料、会议记录或临时想法。

导入的目标不是“保存原文”，而是判断它能不能变成未来 agent 可调用的判断、方法、偏好、案例或复盘线索。

如果用户不希望显式导入材料，优先走 [[_系统/隐式材料收件箱与夜间微信复盘规则|隐式材料收件箱与夜间微信复盘规则]]：白天只收候选，晚上复盘后再写回。

## 第一步：保留来源

先记录来源类型：

- Codex 或 GPT 对话。
- 文档或网页。
- 项目材料。
- 用户自己的想法。
- 外部建议。
- 任务结果。

来源要能帮助后续判断可信度，但不必保存完整原文。

## 第二步：判断材料命运

新内容只有五种命运：

- 丢弃：没有长期价值。
- 收件箱暂存：价值不确定，需要以后合并判断。
- 补旧卡：能增强已有方法、偏好、案例或规则。
- 新建卡：暴露了新的上位问题。
- 进入复盘：能帮助改进下一次行动。

默认优先级是：补旧卡高于新建卡。

## 第三步：先路由再写入

导入前先看：

- [[90_索引/Agent 启动上下文|Agent 启动上下文]]
- [[90_索引/问题到方法群路由|问题到方法群路由]]
- [[90_索引/方法复用验证队列|方法复用验证队列]]

如果新内容能验证队列里的方法，优先补证据。

如果新内容只是已有方法的一个例子，补进已有卡。

如果新内容只是原始材料，放进收件箱。

## 第四步：提炼成可调用结构

导入后至少要形成一种可调用结构：

- 判断清单。
- 反模式。
- 适用场景。
- 不适用场景。
- 下次行动步骤。
- 用户偏好。
- Agent 交接信息。
- 方法成熟度证据。

如果提炼不出这些结构，就不进入长期知识层。

## 第五步：更新相关入口

只有当入口真的变化时才更新索引。

可能需要更新：

- [[90_索引/首页|首页]]
- [[90_索引/方法论总索引|方法论总索引]]
- [[90_索引/方法成熟度与证据索引|方法成熟度与证据索引]]
- [[90_索引/方法复用验证队列|方法复用验证队列]]
- [[90_索引/项目案例到方法论索引|项目案例到方法论索引]]

不要因为导入了一条新材料，就自动新增首页入口。

## 导入质量门槛

导入前问：

- 它能指导下一次相似任务吗？
- 它是否只是已有方法的例子？
- 它能补充哪张旧卡？
- 它有没有明确适用边界？
- 它是否值得未来 agent 读取？
- 它是否避免了完整聊天记录和代码块？
