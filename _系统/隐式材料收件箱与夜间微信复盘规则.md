# 隐式材料收件箱与夜间微信复盘规则

## 核心偏好

导入新材料不应该打断工作对话。

工作线程里，Agent 不要显式问“这段要不要导入知识库”，也不要把材料整理流程展示成额外任务。

白天只做隐式收集，晚上通过微信轻量复盘，复盘结束后再决定哪些材料进入长期知识库。

## 什么材料要进收件箱

材料满足下面任一条件，可以先进入 [[00_收件箱/夜间复盘候选|夜间复盘候选]]：

- 用户贴了一段外部回答、文档、聊天记录或文章摘录。
- 用户表达了新的长期偏好。
- 某次任务暴露了反复出现的卡点。
- 某个项目出现了可迁移判断、反模式或边界条件。
- 用户说“这个后面可能有用”“这个值得记一下”“下次还会遇到”。
- Agent 发现这段内容可能能补旧方法卡，而不是新建浅卡。
- 工作对话中出现了值得晚上复盘的一句话。

## 什么不要进收件箱

下面内容默认不进入收件箱：

- 一次性路径、端口、临时命令或现场状态。
- 密码、密钥、token、个人隐私和敏感凭证。
- 完整原始聊天记录。
- 只对当前任务有效的执行细节。
- 没有迁移价值的情绪、寒暄和临时判断。
- 已经被明确写入长期方法卡的重复材料。

## 工作对话里的行为

工作对话里，Agent 应该：

- 继续完成当前任务。
- 不显式打断用户问是否导入。
- 不在交付总结里展开复盘。
- 只在必要时把候选材料追加到收件箱。
- 如果不确定，就先放收件箱，不立刻升格为方法论。

工作对话里，Agent 不应该：

- 把知识库整理变成新的可见负担。
- 让用户当场填复盘表。
- 因为想沉淀而降低当前任务推进感。

## 夜间微信复盘触发

夜间复盘优先通过 cc-weixin 进行。

默认触发方式：

- 用户晚上在微信里发送“复盘”。
- Agent 读取 [[00_收件箱/夜间复盘候选|夜间复盘候选]]。
- Agent 只问一个轻问题开始。
- 用户愿意继续时，再展开三问。
- 复盘结束后，Agent 根据 [[_系统/新内容导入 SOP|新内容导入 SOP]] 写回知识库。

如果以后实现主动提醒，需要先确认微信 chat_id 和发送权限，避免误发或打扰。

## 微信复盘的默认开场

默认只问一句：

这次最值得带走的一句话是什么？

如果用户愿意继续，再问：

- 一开始我以为问题是什么？
- 后来发现真正的问题是什么？
- 下次我先做哪一步？

## 复盘结束后的写回规则

复盘结束后，Agent 按顺序判断：

- 是否只清空候选，不进入长期知识库。
- 是否补已有方法卡。
- 是否补偏好卡。
- 是否补方法复用验证队列。
- 是否生成一张轻量复盘卡。
- 是否需要更新 Agent 启动上下文或协作偏好。

长期知识库只保存可迁移判断，不保存完整微信对话。
